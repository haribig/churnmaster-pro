💡

Power BI Desktop connects directly to your running Flask server — no file exports. It polls the live endpoint every time you hit Refresh.

Requires Power BI Desktop (free from microsoft.com/en-us/power-bi)
1

Keep your Flask server running

Open a terminal in your project folder and start the server.

python app.py
2

Power BI Desktop → Home → Get Data → Web

Select Basic and paste this URL. It returns a flat JSON array — one row per prediction.

http://localhost:5000/api/powerbi_live/records

The /records endpoint returns a flat list that Power BI parses directly into a table.

3

In Power Query — Convert to Table → Expand

Power BI shows a List. Click Convert to Table, then click the expand icon on the Column1 header → select all fields → OK. You'll see all columns: id, timestamp, prediction, probability, risk, label, domain, date, hour, customer_id.

4

Build your visuals

Use probability for gauges, risk for slicers, date for trend lines, label for donut charts.

5

Click Refresh = live data pulled instantly

No re-importing files. Every Refresh call re-hits your Flask API and returns the latest predictions from the database.

⚠️

Power BI Service cannot reach localhost — you need a public URL for scheduled refresh.

For local use only: publish from Desktop with static export instead.
1

Expose your server publicly

Fastest — ngrok (free):

ngrok http 5000

You'll get a URL like https://abc123.ngrok.io. Use that instead of localhost in Power BI.

Permanent — deploy to Railway / Render / Azure: free tiers available, gives you a stable HTTPS URL.

2

Set POWERBI_TOKEN for auth

Add to your .env file:

POWERBI_TOKEN=my_secret_token_here

Use the URL with token in Power BI:

https://YOUR_URL/api/powerbi_live/records?token=my_secret_token_here
3

Publish Desktop → Service, then set Scheduled Refresh

In Desktop: File → Publish → Publish to Power BI. In app.powerbi.com, open your dataset → Settings → Scheduled Refresh → Turn On → set interval. Free plan: up to 8× per day (every 3 hours). Pro: up to 48× (every 30 min).

⚡

ChurnIQ Live Dashboard — no Power BI license needed. Runs in the browser, auto-refreshes from your Flask API every 30 seconds.

Works on same network. Share the URL with teammates.
🔄

How it works: The live dashboard fetches /api/powerbi_live every 30 seconds. Charts, KPIs and the predictions table all update automatically.

You can also use this endpoint with Grafana, Tableau, Google Looker Studio, or any BI tool that can read JSON from a URL.
🔑

Token auth for public endpoints

Set POWERBI_TOKEN in .env. Anyone calling without ?token=... gets HTTP 401.

POWERBI_TOKEN=change_to_something_long_and_random
🔒

Use HTTPS in production

Never serve on plain HTTP publicly. Use nginx + Certbot, or a platform with automatic TLS (Railway, Render, Azure App Service).

👔

Enterprise: Power BI Embedded

For per-user Row-Level Security, AAD authentication, and no public embed links, use Power BI Embedded (Azure). Needs an Azure AD app registration + Premium Capacity (~$735/month A1 SKU). Reference: docs.microsoft.com/power-bi/developer/embedded

🖥 Retrain on a Different Dataset — Terminal Commands
# Step 1: Copy your CSV file into the project root folder # Step 2: Open terminal in the project folder and run: # ── Telecom (already trained, default) ─────────────────────────────── python train.py --csv customer_churn_prediction_dataset.csv --target Churn # ── Banking dataset ─────────────────────────────────────────────────── python train.py --csv "Bank Customer Churn Prediction.csv" --target Exited # ── IBM HR Attrition ────────────────────────────────────────────────── python train.py --csv "WA_Fn-UseC_-HR-Employee-Attrition.csv" --target Attrition # ── Any custom CSV (omit --target to auto-detect) ───────────────────── python train.py --csv your_data.csv --target YourChurnColumn # Step 3: After training completes (~2 min), hot-reload without restart curl -X POST http://localhost:5000/api/reload_model
⚠️

After switching datasets, the /predict form fields change (bank features ≠ telecom features). The telecom predict form is pre-built. For other datasets, use the API directly or the ChurnIQ Dashboard's dynamic form.

train.py saves the correct feature_names.pkl — the API and dashboard adapt automatically.
